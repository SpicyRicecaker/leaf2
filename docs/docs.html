Leaf Viewer — codebase intuition guide

A real-time OpenGL leaf animation with a detached frame bar, a live graph window, and a shared thread-safe clock. This document transfers everything you need to confidently modify, extend, or debug the project.

2 · SharedClock

SharedClock is intentionally the simplest possible piece of the system. If you ever feel the urge to add rendering logic or UI state here — don't. Keep it a pure time-source.

Internals — how time works

The clock stores two numbers:

_offset  — the logical time (seconds) at the last anchor point
_base    — the wall-clock time (perf_counter) at the last anchor point

Getting the current time is just:

# playing
return _offset + (perf_counter() - _base)

# paused
return _offset   # frozen snapshot

Every write operation re-anchors both values together. This is why seeking while paused, then resuming, works correctly — _base is always reset to now so no phantom time accumulates.

Common Misconception
It might look like pause() and then set_time(t) followed by resume() could drift. It cannot — set_time resets _base to perf_counter() unconditionally, so resume always starts counting from the exact moment set_time was called.

Configuration Constants

The Scroll System — Three Layers

Understanding the scroll system is the most important thing for maintaining this file. There are three separate mechanisms that all modify self._scroll:

Layer 1 — Auto-centre (playback driven)

# fires every poll() tick, ONLY when _manual_scroll is False
if frame >= self._scroll + VIEWPORT_FRAMES - 1:
    self._scroll = frame - VIEWPORT_FRAMES // 2

This is the "DAW playhead" behaviour. It keeps the playhead visible by recentring the viewport when the playhead would scroll off the right edge. It is suppressed by the _manual_scroll flag.

Layer 2 — Manual scroll (user driven)

Set _manual_scroll = True when: scrollbar thumb drag, scrollbar track click, mouse wheel. Clear it when: play is resumed, reset is called.

The Bug This Flag Exists to Prevent
Without _manual_scroll, this sequence breaks:
  1. User drags scrollbar thumb far left
  2. User releases thumb — _sb_state returns to IDLE
  3. Next poll() tick runs — auto-centre fires — _scroll jumps back to playhead
  4. User clicks a cell — _canvas_x_to_frame uses the new (wrong) _scroll
  5. Wrong frame is selected despite correct visual position
The flag keeps _scroll stable until the user explicitly hands control back to playback by pressing play.

Layer 3 — Edge-scroll (drag boundary)

When the mouse is held down on the frame canvas and moves within EDGE_ZONE (30px) of either edge, a repeating after() job fires every EDGE_INTERVAL_MS ms, incrementing _scroll and calling set_frame() simultaneously so the seek and the scroll stay in sync.

Tip
The edge-scroll job stores its after() id in self._edge_job. Always cancel it via _cancel_edge_scroll() — never by setting the id to None directly — because the cancellation also resets _edge_direction. Missing that reset causes the direction guard to break on the next drag.

4 · main.py

The Render Loop

while running:
    dt = clock_obj.tick(0) / 1000.0   # pygame wall-clock delta, uncapped
    dt = min(dt, 0.05)                # spiral-of-death guard (max 50ms)
    t  = clock.get_time()             # SharedClock logical time

    # events → keyboard → model matrix → draw
Important Distinction
There are two clocks in main.py:
  • clock_obj — a pygame.time.Clock(). Used only to compute dt for camera movement. Has nothing to do with the scene time.
  • clock — the SharedClock. Drives everything visible: leaf position, framebar, graph.
Never use pygame.time.get_ticks() for scene time after this refactor.

Keyboard Routing

Keyboard shortcuts that affect SharedClock are handled in two places and must be kept in sync:

Keymain.py (KEYDOWN event)framebar.py (root.bind)
Spaceclock.toggle_pause()self._toggle_pause()
→clock.step_frames(1)self._step(1)
←clock.step_frames(-1)self._step(-1)
Rclock.reset()self._reset()
Watch Out
framebar._reset() does more than clock.reset() — it also resets _scroll, _hwm, _total_frames, and _manual_scroll. Pressing R in the pygame window calls only clock.reset(), so the framebar viewport does not reset. This is a known minor inconsistency. If you want full reset from both windows, expose a full_reset() method on FrameBar and call it from main.py.

Leaf Transform

model = T * R    (pyrr convention: rightmost applied first)

R = Rz * Ry * Rx    (X applied first, then Y, then Z)

angle = sin(2π t)   # oscillates -1 … +1 radians, period = 1 second
y     = lerp(LEAF_START_Y, LEAF_END_Y, (t % FALL_DURATION) / FALL_DURATION)

The rotation is the same angle on all three axes simultaneously — this produces the characteristic tumbling/fluttering look. To make the leaf spin faster increase the 2π multiplier. To make it fall faster decrease FALL_DURATION.

Tip — pyrr matrix multiply order
pyrr.matrix44.multiply(A, B) returns A × B. Vertices are column vectors, so the rightmost matrix is applied first. multiply(rotation, translation) means: rotate, then translate. This is the opposite of what you might expect if you are used to writing T * R left-to-right.

5 · RealtimeGraph

The graph is the simplest of the three windows. It has no write path — it only reads get_time() and maps it to a position on y = x via a triangle wave. 

# x oscillates 0 → 1 → 0, period = 4 seconds
phase = (t % 4.0) / 4.0
x = phase * 2        if phase < 0.5
x = (1 - phase) * 2  otherwise
y = x

To plot a different function: change y = x to whatever you want in _run(). To change the axis range: update ax.set_xlim / ax.set_ylim. To change the oscillation period: change the 4.0 in _compute_x.

Note — matplotlib backend
The file explicitly sets matplotlib.use("TkAgg") before importing pyplot. This must happen before any other matplotlib import. On some systems TkAgg is not available — try Qt5Agg or WXAgg as fallbacks.

6 · Common Bugs & Misconceptions

Scroll / Seek Bugs

SymptomCauseFix
Clicking a cell seeks to the wrong frame after scrolling _scroll was modified by auto-centre between the scroll action and the click Ensure _manual_scroll = True was set at scroll time and not prematurely cleared
Auto-centre never re-engages after manual scroll _manual_scroll never got cleared back to False Check that _toggle_pause clears it on resume, and that _reset clears it
Edge-scroll keeps firing after mouse release _cancel_edge_scroll() not called on ButtonRelease-1 Always call cancel in _cell_release
Scrollbar thumb jumps on first drag _sb_drag_start_scroll not captured at press time Capture it in _sb_press, not _sb_drag

Threading Pitfalls

Never Do This
# WRONG — calling tkinter from the main thread
self._frame_label.config(text="...")   # from main.py or graph.py

# WRONG — calling pygame from the framebar thread
pygame.display.flip()   # from framebar.py

# WRONG — reading SharedClock without going through its methods
t = clock._offset   # bypasses the lock

Tkinter is not thread-safe. All tkinter calls must happen on the thread that called mainloop(). The correct pattern for cross-thread communication into tkinter is root.after(0, callback) — schedules the callback to run on the tkinter thread on the next event loop iteration.

Safe Cross-Thread → Tkinter Pattern
# from any thread:
self._root.after(0, lambda: self._frame_label.config(text="hello"))

OpenGL Gotchas

SymptomCauseNotes
Black screen, no errors Shader compilation failed silently, or VAO not bound before draw Check glGetProgramInfoLog(program) after linking
Leaf is one solid colour Normal attribute (location 1) not set up, lighting returns 0 Check glVertexAttribPointer offset for normals = 3 floats = 12 bytes
Leaf only shows one side GL_CULL_FACE enabled — back face culled The codebase explicitly calls glDisable(GL_CULL_FACE) — make sure it stays
Matrix uniform does nothing Uniform name typo, or glUseProgram not called before glUniform Always call glUseProgram(program) before any uniform upload
GL_INVALID_OPERATION on draw VAO unbound or wrong VAO bound glBindVertexArray(vao) immediately before glDrawArrays
pyrr matrix layout
pyrr matrices are row-major in memory but OpenGL expects column-major. glUniformMatrix4fv(..., GL_FALSE, matrix) — the GL_FALSE means "do not transpose". This works correctly with pyrr because pyrr's create_look_at and create_perspective_projection_matrix already produce the layout OpenGL wants. If you swap to a different math library, verify the transpose flag.

7 · Extending the Codebase

Adding a New Secondary Window

Follow this checklist:

  1. Create a class with a start() method that spawns a daemon=True thread
  2. The thread's entry point calls the GUI toolkit's blocking loop (mainloop(), plt.show(), etc.)
  3. Accept a SharedClock in __init__ — never accept the raw time value
  4. Only read/write SharedClock via its public methods
  5. Never import pygame or call any OpenGL function from the new thread
  6. Instantiate and call .start() in main() after pygame init but before the main loop

Adding a New Shader Uniform

# 1. Declare in the shader
uniform float myValue;

# 2. Get location once, after build_program()
u_myValue = glGetUniformLocation(program, "myValue")

# 3. Upload every frame inside the render loop
glUseProgram(program)   # must be active
glUniform1f(u_myValue, some_python_float)

Changing the Frame Rate Definition

FRAME_MS in shared_clock.py is the only place to change. Everything derives from it: 

FRAME_DURATION = FRAME_MS / 1000.0

get_frame()     = int(get_time() / FRAME_DURATION)
set_frame(n)    = set_time(n * FRAME_DURATION)
step_frames(d)  = move by d * FRAME_DURATION seconds

The framebar cell size in pixels does not change — each cell always represents exactly one frame regardless of how long a frame is in wall-clock time.

Common Frame Rate Values
24 fps  →  FRAME_MS = 41.6667
30 fps  →  FRAME_MS = 33.3333
60 fps  →  FRAME_MS = 16.6667
120 fps →  FRAME_MS =  8.3333
